# Guide Atelier 5 - Feuille de Route

## Objectif
Finaliser le dossier EBIOS RM et construire la feuille de route stratégique.

---

## 🎯 Livrables attendus

1. **Feuille de route** détaillée (plan d'action 1-3 ans)
2. **Synthèse pour la direction** (présentation exécutive)
3. **Dossier EBIOS RM complet** (tous les ateliers)

---

## ⏱️ Durée et participants

- **Durée** : 3 à 4 heures
- **Participants** :
  - Sponsor / Direction (validation finale)
  - RSSI / Responsable sécurité
  - MOA (validation plan d'action)
  - Pilotes des chantiers (engagement)

---

## 📋 Déroulé de l'atelier

### Étape 1 : Rappel et synthèse (30 min)

**Rappel des 4 ateliers précédents** :
- Atelier 1 : [X] biens essentiels identifiés
- Atelier 2 : [X] sources de risque
- Atelier 3 : [X] scénarios de risque, [Y] risques critiques
- Atelier 4 : [X] mesures de sécurité proposées

**Synthèse des résultats clés** :
- Top 5 des risques les plus graves
- Mesures prioritaires identifiées
- Investissement global estimé

---

### Étape 2 : Construction de la feuille de route (1h30)

**Objectif** : Organiser les actions sur 1 à 3 ans

#### Vision globale

| Horizon | Objectif | Actions types |
|---------|----------|---------------|
| **Court terme** (0-6 mois) | Sécuriser l'urgent | Quick wins, mesures critiques |
| **Moyen terme** (6-18 mois) | Structurer | Projets importants, outils |
| **Long terme** (18-36 mois) | Maturité | Transformation, culture |

#### Structure de la feuille de route

Pour chaque action :

| Champ | Description |
|-------|-------------|
| ID | Référence unique (FR-001) |
| Action | Description détaillée |
| Objectif | Quel risque traite-t-elle ? |
| Horizon | Court / Moyen / Long terme |
| Début | Date de début prévue |
| Fin | Date de fin prévue |
| Budget | Montant estimé |
| Responsable | Pilote nommé |
| Contributeurs | Équipe impliquée |
| Indicateurs | Comment mesurer le succès ? |
| Dépendances | Actions préalables |

#### Exemple de feuille de route

| ID | Action | Horizon | Début | Fin | Budget | Responsable |
|----|--------|---------|-------|-----|--------|-------------|
| FR-001 | Déploiement MFA | Court | 01/03 | 30/04 | 50K€ | RSSI |
| FR-002 | Projet SIEM | Moyen | 01/06 | 31/12 | 200K€ | RSI |
| FR-003 | Programme sensibilisation | Long | 01/01 | 31/12 | 100K€ | RH |

---

### Étape 3 : Gouvernance de suivi (30 min)

**Objectif** : Définir comment suivre la feuille de route

#### Comité de pilotage

| Élément | Description |
|---------|-------------|
| Fréquence | Mensuelle / Trimestrielle |
| Participants | RSSI, MOA, Pilotes |
| Ordre du jour | Avancement, blocages, ajustements |
| Reporting | Tableau de bord sécurité |

#### Indicateurs de suivi (KPI)

| KPI | Cible | Fréquence |
|-----|-------|-----------|
| % mesures déployées | 100% | Mensuelle |
| Budget consommé / prévu | ±10% | Mensuelle |
| Risques réduits | [Nombre] | Trimestrielle |
| Incidents de sécurité | [Objectif] | Mensuelle |

#### Revue annuelle EBIOS

- **Fréquence** : Annuelle minimum
- **Déclencheurs** : Incident majeur, changement significatif
- **Contenu** : Mise à jour des risques, ajustement feuille de route

---

### Étape 4 : Présentation direction (30 min)

**Objectif** : Valider le dossier et obtenir l'engagement

#### Structure de la synthèse direction

**1. Contexte (1 slide)**
- Périmètre de l'étude
- Méthodologie EBIOS RM

**2. Constats (1-2 slides)**
- Biens essentiels critiques
- Principaux risques identifiés
- Écarts de sécurité majeurs

**3. Enjeux (1 slide)**
- Risques si on n'agit pas
- Opportunités si on agit

**4. Plan d'action (2-3 slides)**
- Feuille de route synthétique
- Investissement global
- Calendrier

**5. Demande (1 slide)**
- Validation du plan
- Engagement des ressources
- Prochaines étapes

#### Messages clés pour la direction

- **Le risque cyber est un risque métier** (pas juste IT)
- **L'investissement est nécessaire** (coût vs risque)
- **Le plan est réaliste** (échéances, ressources)
- **La direction est engagée** (gouvernance, suivi)

## 💡 Astuces de facilitation

### Pour la feuille de route
- **Décomposer les gros projets** : étapes tangibles
- **Mentionner les jalons** : points de validation
- **Anticiper les congés** : pas de fin décembre
- **Buffer de sécurité** : 20% de marge sur les délais

### Pour la présentation direction
- **Commencer par le risque** : "Voici ce qui pourrait nous arriver"
- **Parler en euros** : traduire le risque en financier
- **Proposer une solution** : "Voici notre plan pour y répondre"
- **Demander une décision** : "Nous avons besoin de votre validation"

### Pour la finalisation
- **Relire à froid** : attendre 24h avant envoi définitif
- **Faire relire** : par RSSI et MOA
- **Versionner** : V1.0 Final, date

---

## ⚠️ Pièges classiques

| Piège | Solution |
|-------|----------|
| Feuille de route surchargée | Maximum 10-15 actions par an |
| Budget sous-estimé | Ajouter 20% de marge |
| Échéances irréalistes | Valider avec les pilotes |
| Présentation trop technique | Simplifier, focus risque métier |
| Oublier la maintenance | Prévoir budget récurrent |
