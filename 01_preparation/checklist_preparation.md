- [ ] Salle réservée (assez grande pour groupe de travail)
- [ ] Vidéoprojecteur/écran
- [ ] Tableau blanc / paperboard
- [ ] Post-its (plusieurs couleurs)
- [ ] Marqueurs
- [ ] Café/pause prévues

### 2.3 Outils digitaux (si visio)
- [ ] Lien visio testé (Teams, Zoom, Meet)
- [ ] Outil de tableau blanc (Miro, Mural, Menti)
- [ ] Partage d'écran fonctionnel
- [ ] Document collaboratif (Google Docs, Confluence)

---

## 3. DOCUMENTS À DEMANDER AU CLIENT

### 3.1 Documents stratégiques
- [ ] Organigramme général
- [ ] Organigramme de la sécurité (RSSI, équipe)
- [ ] Politique de sécurité des systèmes d'information
- [ ] Charte informatique
- [ ] PSSI (Politique de Sécurité des SI) si existe

### 3.2 Documents techniques
- [ ] Schéma d'architecture réseau
- [ ] Schéma d'architecture applicative
- [ ] Inventaire des systèmes (CMDB)
- [ ] Cartographie des flux
- [ ] Registre des traitements RGPD

### 3.3 Documents opérationnels
- [ ] PCA (Plan de Continuité d'Activité) si existe
- [ ] PRA (Plan de Reprise d'Activité) si existe
- [ ] Procédure de gestion des incidents
- [ ] Rapports d'audit précédents
- [ ] Synthèse des incidents passés

### 3.4 Contexte métier
- [ ] Description des processus métier critiques
- [ ] Cartographie des fournisseurs critiques
- [ ] Exigences réglementaires du secteur
- [ ] Contrats avec exigences sécurité

---

## 4. OUTILS À PRÉPARER

### 4.1 Fichier EBIOS RM
- [ ] Créer le fichier de travail (format .ebios ou équivalent)
- [ ] Configurer le périmètre
- [ ] Pré-remplir les informations connues

### 4.2 Templates
- [ ] Template de compte-rendu d'atelier
- [ ] Grille d'évaluation des biens essentiels
- [ ] Fiche de scénario de risque
- [ ] Template de feuille de route

### 4.3 Référentiels
- [ ] Guide EBIOS RM ANSSI (dernière version)
- [ ] ISO 27002 (mesures de sécurité)
- [ ] MITRE ATT&CK (tactiques d'attaque)
- [ ] Référentiel sectoriel si applicable

---

## 5. PRÉPARATION PERSONNELLE

### 5.1 Revue des documents
- [ ] Lire la documentation du client
- [ ] Identifier les points de vigilance
- [ ] Préparer des questions ciblées

### 5.2 Scénarios types du secteur
- [ ] Identifier les menaces sectorielles courantes
- [ ] Préparer des exemples de sources de risque
- [ ] Avoir des scénarios de référence

### 5.3 Anticiper les difficultés
- [ ] Quels sujets sensibles ?
- [ ] Quels participants difficiles ?
- [ ] Quels désaccords potentiels ?
- [ ] Quel niveau de maturité sécurité ?

---

## 6. CHECKLIST JOUR J (avant chaque atelier)

### 2 heures avant
- [ ] Matériel prêt (post-its, marqueurs, paperboard)
- [ ] Documents imprimés (si nécessaire)
- [ ] Présentation/projets testés
- [ ] Salle préparée / connexion visio testée

### 30 minutes avant
- [ ] Arriver en avance / se connecter en avance
- [ ] Vérifier le matériel technique
- [ ] Préparer l'accueil des participants
- [ ] Se mettre en condition (zen, concentré)

---

## 7. POST-ATELIER

### Immédiat (même jour)
- [ ] Photos des tableaux/post-its
- [ ] Sauvegarde documents digitaux
- [ ] Notes complétées

### Dans les 48h
- [ ] Compte-rendu rédigé
- [ ] Envoi au client pour validation
- [ ] Mise à jour du fichier EBIOS
- [ ] Planification atelier suivant confirmée
