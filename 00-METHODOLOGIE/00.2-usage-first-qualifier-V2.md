## ❓ Questions de qualification (cocher les cases)

### Q1 : L'IA agit-elle sur des données ou processus métier ?
*(vs. simple conversation / brainstorming)*

```yaml
☐ Non → Usage conversationnel uniquement
    Ex: Rédaction d'emails, synthèse documentaire, veille
    
☐ Oui → Aller à Q2
    Ex: RAG métier, aide au tri, génération de rapports structurés
```

### Q2 : La sortie de l'IA influence-t-elle une décision ou une action ?
*(vs. aide informative sans conséquence)*

```yaml
☐ Non → Aide informative
    Ex: Suggestions de reformulation, résumé pour lecture humaine
    
☐ Oui → Aller à Q3
    Ex: Pré-sélection de candidats, scoring de tickets, génération de réponses auto
```

### Q3 : La décision/action est-elle automatisée ou à fort impact ?
*(vs. validation humaine systématique)*

```yaml
☐ Non → Validation humaine systématique
    Ex: L'humain valide chaque sortie avant envoi / action
    
☐ Oui → Aller à Q4
    Ex: L'IA agit seule, ou l'humain ne valide que par échantillonnage
```

### Q3bis : Réversibilité de la décision (si Q3=Oui)

```yaml
☐ Réversible immédiatement (sans conséquence)
☐ Réversible avec effort (correction nécessaire)
☐ Irréversible ou conséquences durables → 🔴 recommandé
```

### Q4 : L'usage concerne-t-il un domaine "haut risque" (AI Act Annex III) ?

**Domaines concernés :**

```yaml
☐ Recrutement / sélection de personnel
☐ Évaluation de performance / promotion
☐ Accès à l'éducation, crédit, assurance, santé
☐ Application de la loi, justice, frontière
☐ Identification biométrique à distance
☐ Manipulation subliminale, scoring social
☐ Non → Usage opérationnel standard
☐ Oui → Usage décisionnel / réglementaire ⚠️
```

### Q5 : Des données sensibles sont-elles manipulées ?

```yaml
☐ Publiques / génériques uniquement
☐ Internes non sensibles
☐ Données personnelles (RGPD) → 📌 Référence AIPD requise
☐ Données critiques (santé, financier, secret) → 📌 Niveau 🔴 recommandé
```

## 🎯 Classification finale

### ➤ Si Q1 = Non
🟢 **NIVEAU LIGHT** — Usage Conversationnel
- Template à utiliser : `fiche-light.md`
- Effort : 10-15 min, pas d'atelier formel
- Exemples : Chat, rédaction, brainstorming, veille

### ➤ Si Q1=Oui ET Q2=Non (ou Q3=Non)
🟡 **NIVEAU STANDARD** — Usage Workflow / Semi-Automatisé
- Template à utiliser : `fiche-workflow.md`
- Effort : Atelier 1h30-2h, équipe pluridisciplinaire
- Exemples : RAG métier, aide au tri avec validation, génération structurée

### ➤ Si Q1=Oui ET Q2=Oui ET Q3=Oui (ou Q4=Oui)
🔴 **NIVEAU RENFORCÉ** — Usage Agentique / Décisionnel
- Template à utiliser : `dossier-decisionnel.md`
- Effort : Projet 1-2 jours en itératif, gouvernance formelle
- Exemples : Tri automatique, scoring, actions autonomes, usage Annex III

## 🔄 Cas particuliers & règles d'interprétation

### ⚠️ Règle de prudence
> En cas de doute entre deux niveaux, partez du niveau supérieur et allégez ensuite si l'analyse le permet.

### 🔄 Évolution d'usage
> Un usage peut changer de niveau dans le temps. Ex: un chatbot 🟢 devient 🟡 si ses sorties sont intégrées automatiquement dans un CRM.

### 🧩 Usage composite
> Si un même modèle sert plusieurs usages : qualifier chaque usage séparément. Ex: Qwen3.5 peut être 🟢 pour la rédaction et 🔴 pour le scoring.

### 📦 Variante technique (LoRA, SFT)
> Le fine-tuning ne change pas automatiquement le niveau. C'est l'usage final qui prime. Mais : documenter la variante dans le template choisi.
